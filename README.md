# Team Productivity Tracker

This project simulates a professional Git and GitHub workflow in a team environment.

## Goal
Demonstrate:
- Branching strategy (feature branches)
- Pull requests and basic code review
- Issue tracking
- Merge conflicts and conflict resolution
- Clear repository documentation

## Team Workflow (Summary)
1. Create an Issue for each task
2. Create a feature branch from main
3. Commit changes with clear messages
4. Open a Pull Request (PR)
5. Review and merge the PR
6. Resolve conflicts if they appear
